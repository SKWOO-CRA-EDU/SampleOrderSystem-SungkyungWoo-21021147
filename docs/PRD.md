# PRD — 반도체 시료 생산 주문관리 시스템

> 본 문서는 `docs/SPEC.md`와 `docs/DECISIONS.md`만을 근거로 작성되었다.
> 각 요구사항 옆에 근거를 `[SPEC §x]` 또는 `[ADR-Qn]`으로 표기하며, 근거 없는 요구사항은 포함하지 않는다.
> 두 문서에서 근거를 찾지 못한 항목은 "OPEN"으로 표시하고 임의로 결정하지 않는다.

---

## 1. 개요 / 사용자 역할

가상의 반도체 회사 "S-Semi"는 시료(Sample)를 생산하여 고객에게 납품한다. 엑셀/메모장 기반 수기 관리로 인한 처리 누락, 재고·공정 파악 곤란 문제를 해결하기 위해 콘솔 기반 주문관리 시스템을 개발한다. [SPEC §1]

| 역할 | 내부/외부 | 담당 업무 |
|---|---|---|
| 고객 | 외부 | 이메일로 시료 요청 (시스템 외부 채널, 시스템 사용자 아님) [SPEC §2] |
| 주문담당자 | 내부 | 요청에 맞는 주문서 작성(주문 접수), 생산담당자에게 전달 [SPEC §2] |
| 생산담당자 | 내부 | 시료(개발시료) 등록, 주문 수신 후 승인/거절 [SPEC §2] |

시스템은 콘솔 기반으로 동작하며 담당자가 직접 명령을 입력해 시료 등록과 주문 처리를 수행한다. [SPEC §7-콘솔]
본 시스템은 단일 프로세스 실행을 전제로 하며, 동일 데이터 저장소에 대한 멀티 프로세스 동시 접근은 지원 범위에서 제외한다. [ADR-Q10]

## 2. 도메인 모델

### 2.1 Sample (시료)

| 속성 | 타입/제약 | 근거 |
|---|---|---|
| 시료ID | 고유 식별자 | [SPEC §5-1] |
| 이름 | 문자열 | [SPEC §5-1] |
| 평균생산시간 | 양수(> 0) | [SPEC §5-1], [ADR-Q12] |
| 수율 | (0, 1] 범위의 실수 | [SPEC §4], [ADR-Q4] |
| 재고수량 | 음이 아닌 정수(≥ 0), 시스템 내부 로직(승인 시 차감, 생산완료 시 증가)에 의해서만 변경 | [ADR-Q1], [ADR-Q9], [ADR-Q12] |

**불변식**
- 수율 ∈ (0, 1] — 등록 시점 검증으로 강제, 벗어나는 값은 등록 거부. [ADR-Q3], [ADR-Q4]
- 평균생산시간 > 0 — 등록 시점 검증. [ADR-Q12]
- 재고수량 ≥ 0 — 항상 유지되는 불변식(코드 레벨 보장). [ADR-Q1], [ADR-Q12]

### 2.2 Order (주문)

| 속성 | 타입/제약 | 근거 |
|---|---|---|
| 주문ID | 자동증가 정수(1, 2, 3, ...) | [ADR-Q15] |
| 시료ID | 등록된 시료 참조 | [SPEC §5-2] |
| 고객명 | 문자열 | [SPEC §5-2] |
| 주문수 | 양의 정수(> 0) | [SPEC §5-2], [ADR-Q12] |
| 상태 | RESERVED / REJECTED / PRODUCING / CONFIRMED / RELEASE | [SPEC §3] |

### 2.3 생산큐 항목 (ProductionQueueItem)

| 속성 | 의미 | 근거 |
|---|---|---|
| 대상 주문 | PRODUCING 상태로 전이된 주문 | [SPEC §3, §5-3] |
| 부족분 | max(0, 주문수 − 재고수량) | [ADR-Q1] |
| 실생산량 | ceil(부족분 / 수율) | [SPEC §4] |
| 총생산시간 | 평균생산시간 × 실생산량 | [SPEC §4], [ADR-Q16] |
| 큐 내 순서 | FIFO(등록 순) | [SPEC §5-5], [ADR-Q11] |

**불변식**
- 생산큐는 단일 생산라인을 전제로 하며, 한 번에 하나의 항목만 실행 상태로 처리된다(순차 생산). [ADR-Q11]

## 3. 상태 전이 명세

| # | From | To | 트리거 | 근거 |
|---|---|---|---|---|
| T1 | RESERVED | CONFIRMED | 주문승인 — 재고가 주문수 이상(충분) | [SPEC §3, §5-3], [ADR-Q9] |
| T2 | RESERVED | PRODUCING | 주문승인 — 재고가 주문수 미만(부족), 생산라인 자동 등록 | [SPEC §3, §5-3], [ADR-Q9] |
| T3 | RESERVED | REJECTED | 주문거절 | [SPEC §3, §5-3] |
| T4 | PRODUCING | CONFIRMED | 생산완료 | [SPEC §3, §5-5, §6] |
| T5 | CONFIRMED | RELEASE | 출고처리 | [SPEC §3, §5-6, §6] |
| T6 | CONFIRMED | REJECTED | 출고 전 취소, 재고 복구 | [ADR-Q7] |

**지원하지 않는 전이**
- PRODUCING → REJECTED(생산 중 취소): 미지원. [ADR-Q6]
- RELEASE → (모든 전이, 반품/재입고): 미지원. RELEASE는 종단 상태. [ADR-Q8]
- REJECTED → (모든 전이): 미지원. REJECTED는 종단 상태. [ADR-Q8]

### 3.1 T1: RESERVED → CONFIRMED (재고충분 승인)

- **사전조건**: 주문 상태 = RESERVED; 해당 시료 재고수량 ≥ 주문수. [SPEC §3, §5-3]
- **사후조건**: 재고수량 -= 주문수; 주문 상태 = CONFIRMED. [ADR-Q9]

### 3.2 T2: RESERVED → PRODUCING (재고부족 승인)

- **사전조건**: 주문 상태 = RESERVED; 해당 시료 재고수량 < 주문수. [SPEC §3, §5-3]
- **사후조건**: 재고수량 -= 주문수(재고수량이 음수가 될 수 있는 예약 차감, 이후 T4에서 생산량만큼 증가); 주문 상태 = PRODUCING; 생산큐에 FIFO로 등록. [ADR-Q9], [ADR-Q11]

  > OPEN: 재고수량 불변식(§2.1, ≥0)과 T2 사후조건(차감으로 인한 일시적 음수 가능성)의 관계는 SPEC/DECISIONS에 명시적으로 정리되어 있지 않다. ADR-Q9는 "즉시 차감"을, ADR-Q1/Q12는 "재고수량 ≥ 0 불변식"을 각각 규정하나 양자가 동시에 성립하는 구간(차감 직후 ~ 생산완료 전)에 대한 명시적 조정 규칙은 없다. 별도 확인 필요.

### 3.3 T3: RESERVED → REJECTED (거절)

- **사전조건**: 주문 상태 = RESERVED. [SPEC §3, §5-3]
- **사후조건**: 주문 상태 = REJECTED. 재고수량 변경 없음(T3 이전에는 아직 차감이 발생하지 않았으므로). [ADR-Q9]

### 3.4 T4: PRODUCING → CONFIRMED (생산완료)

- **사전조건**: 주문 상태 = PRODUCING; 해당 생산큐 항목의 생산이 완료됨(총생산시간 경과). [SPEC §5-5, §6]
- **사후조건**: 재고수량 += floor(실생산량 × 수율); 주문 상태 = CONFIRMED. [ADR-Q13], [ADR-Q9]

### 3.5 T5: CONFIRMED → RELEASE (출고)

- **사전조건**: 주문 상태 = CONFIRMED; 재고가 충분해진 상태(출고 시점 재검증). [SPEC §5-6, §6], [ADR-Q9]
- **사후조건**: 주문 상태 = RELEASE. [SPEC §5-6, §6]

### 3.6 T6: CONFIRMED → REJECTED (출고 전 취소)

- **사전조건**: 주문 상태 = CONFIRMED. [ADR-Q7]
- **사후조건**: 재고수량 += 주문수(승인 시 차감분 복구); 주문 상태 = REJECTED. [ADR-Q7]

## 4. 기능 요구사항

### FR-01: 시료등록

시스템은 시료ID, 이름, 평균 생산시간, 수율을 입력받아 새로운 시료를 등록할 수 있어야 한다. [SPEC §5-1]

- **Given** 등록된 적 없는 시료ID와 유효한 이름, 평균생산시간(> 0), 수율(0 < 수율 ≤ 1)을 입력했을 때
- **When** 시료등록을 실행하면
- **Then** 새로운 시료가 재고수량 0으로 시스템에 추가된다.

### FR-02: 시료 유효성 검증 — 수율 범위

시스템은 수율이 (0, 1] 범위를 벗어나면 시료등록을 거부해야 한다. [ADR-Q3], [ADR-Q4]

- **Given** 수율 값이 0이거나 1을 초과하는 입력
- **When** 시료등록을 시도하면
- **Then** 등록이 거부되고 시료가 저장되지 않는다.

### FR-03: 시료 유효성 검증 — 평균생산시간

시스템은 평균생산시간이 0 이하이면 시료등록을 거부해야 한다. [ADR-Q12]

- **Given** 평균생산시간 값이 0 또는 음수인 입력
- **When** 시료등록을 시도하면
- **Then** 등록이 거부되고 시료가 저장되지 않는다.

### FR-04: 시료조회

시스템은 등록된 모든 시료 목록과 각 시료의 현재 재고수량을 표시해야 한다. [SPEC §5-1]

- **Given** 하나 이상의 시료가 등록되어 있을 때
- **When** 시료조회를 실행하면
- **Then** 모든 시료의 목록과 각 재고수량이 함께 표시된다.

### FR-05: 시료검색

시스템은 시료ID, 이름, 수율, 평균생산시간을 대상으로 한 검색어로 등록된 시료를 검색할 수 있어야 한다. [SPEC §5-1], [ADR-Q14]

- **Given** 등록된 시료 중 검색어와 일치하는 시료ID/이름/수율/평균생산시간이 존재할 때
- **When** 시료검색을 실행하면
- **Then** 일치하는 시료(재고수량 포함)만 결과로 표시된다.

### FR-06: 시료예약(주문 생성)

시스템은 시료ID, 고객명, 주문수를 입력받아 새 주문을 RESERVED 상태로 생성해야 한다. [SPEC §5-2]

- **Given** 등록된 시료ID, 고객명, 주문수(> 0)를 입력했을 때
- **When** 시료예약을 실행하면
- **Then** 자동증가 주문ID가 부여된 RESERVED 상태의 주문이 생성된다.

### FR-07: 주문수 유효성 검증

시스템은 주문수가 0 이하이면 시료예약을 거부해야 한다. [ADR-Q12]

- **Given** 주문수가 0 또는 음수인 입력
- **When** 시료예약을 시도하면
- **Then** 예약이 거부되고 주문이 생성되지 않는다.

### FR-08: 접수된 주문목록 표시

시스템은 RESERVED 상태인 주문 목록을 표시해야 한다. [SPEC §3, §5-3]

- **Given** RESERVED 상태의 주문이 하나 이상 존재할 때
- **When** 접수된 주문목록 조회를 실행하면
- **Then** RESERVED 상태 주문만 목록으로 표시된다.

### FR-09: 주문승인 — 재고충분

시스템은 RESERVED 주문 승인 시 대상 시료 재고수량이 주문수 이상이면 즉시 CONFIRMED로 전환해야 한다. [SPEC §3, §5-3], [ADR-Q9]

- **Given** RESERVED 상태 주문의 시료 재고수량 ≥ 주문수일 때
- **When** 해당 주문을 승인하면
- **Then** 재고수량에서 주문수만큼 차감되고 주문 상태가 CONFIRMED로 전환된다.

### FR-10: 주문승인 — 재고부족

시스템은 RESERVED 주문 승인 시 대상 시료 재고수량이 주문수 미만이면 생산라인(생산큐)에 자동 등록하고 PRODUCING으로 전환해야 한다. [SPEC §3, §5-3], [ADR-Q9], [ADR-Q11]

- **Given** RESERVED 상태 주문의 시료 재고수량 < 주문수일 때
- **When** 해당 주문을 승인하면
- **Then** 주문 상태가 PRODUCING으로 전환되고 해당 주문이 생산큐의 FIFO 순서 끝에 등록된다.

### FR-11: 주문거절

시스템은 RESERVED 주문을 즉시 REJECTED로 전환할 수 있어야 한다. [SPEC §3, §5-3]

- **Given** RESERVED 상태의 주문
- **When** 해당 주문을 거절하면
- **Then** 주문 상태가 REJECTED로 전환된다.

### FR-12: CONFIRMED 주문 취소

시스템은 CONFIRMED 상태 주문을 출고 전 취소(REJECTED)할 수 있어야 하며, 취소 시 승인 단계에서 차감된 재고를 복구해야 한다. [ADR-Q7]

- **Given** CONFIRMED 상태의 주문
- **When** 해당 주문을 취소하면
- **Then** 재고수량에 주문수만큼 복구되고 주문 상태가 REJECTED로 전환된다.

### FR-13: PRODUCING 주문 취소 미지원

시스템은 PRODUCING 상태 주문에 대한 거절/취소 기능을 제공하지 않는다. [ADR-Q6]

- **Given** PRODUCING 상태의 주문
- **When** 해당 주문에 대해 거절을 시도하면
- **Then** 시스템은 해당 전이를 지원하지 않는다(기능 자체가 제공되지 않음).

### FR-14: 생산라인 정보 표시

시스템은 생산라인에 대한 정보를 표시해야 한다. [SPEC §3, §5-5, §6]

- **Given** 생산큐에 하나 이상의 항목이 존재할 때
- **When** 생산 라인 메뉴를 조회하면
- **Then** 생산라인 관련 정보가 표시된다.

### FR-15: 생산현황표기

시스템은 현재 생산 중인 시료에 대한 정보를 표시해야 한다. [SPEC §5-5, §6]

- **Given** PRODUCING 상태로 생산 진행 중인 항목이 있을 때
- **When** 생산현황표기를 조회하면
- **Then** 현재 생산 중인 항목의 정보가 표시된다.

> OPEN: 표기할 정보 항목의 구체적 수준은 SPEC에서 "자율적으로 결정하도록" 명시되어 있어(§5-5, §6) 본 PRD는 항목을 확정하지 않는다. [SPEC §5-5, §6]

### FR-16: 대기주문확인

시스템은 생산큐(FIFO)에서 생산을 대기 중인 주문 목록을 표시해야 한다. [SPEC §5-5], [ADR-Q11]

- **Given** 생산큐에 대기 중인 항목이 하나 이상 있을 때
- **When** 대기주문확인을 실행하면
- **Then** 큐에 등록된 순서(FIFO)대로 대기 목록이 표시된다.

### FR-17: 생산 계산 — 실생산량

시스템은 PRODUCING 항목에 대해 실생산량 = ceil(부족분 / 수율)을 계산해야 한다. [SPEC §4], [ADR-Q1], [ADR-Q2]

- **Given** 부족분과 대상 시료의 수율 값이 주어졌을 때
- **When** 실생산량을 계산하면
- **Then** ceil(부족분 / 수율) 값이 산출된다(예: 부족분 170, 수율 0.92 → ceil(184.7826...) = 185).

### FR-18: 생산 계산 — 총생산시간

시스템은 실생산량이 산출된 항목에 대해 총생산시간 = 평균생산시간 × 실생산량을 계산해야 한다. [SPEC §4], [ADR-Q2], [ADR-Q16]

- **Given** 평균생산시간과 실생산량(계산값)이 주어졌을 때
- **When** 총생산시간을 계산하면
- **Then** 평균생산시간 × 실생산량 값이 산출되고, 화면에는 "×" 기호로 표기된다(예: 평균생산시간 0.8, 실생산량 185 → 148).

### FR-19: 생산완료 처리

시스템은 생산이 완료되면 실생산량 × 수율만큼(내림) 재고수량에 반영하고 대상 주문 상태를 PRODUCING → CONFIRMED로 전환해야 한다. [SPEC §3, §5-5, §6], [ADR-Q13]

- **Given** PRODUCING 상태 주문의 생산큐 항목이 총생산시간만큼 처리를 완료했을 때
- **When** 생산완료 처리가 실행되면
- **Then** 재고수량에 floor(실생산량 × 수율)만큼 증가하고, 해당 주문 상태가 CONFIRMED로 전환된다.

### FR-20: 출고처리

시스템은 CONFIRMED 상태이며 재고가 충분해진 주문에 대해 출고를 실행하고 RELEASE로 전환해야 한다. [SPEC §3, §5-6, §6]

- **Given** CONFIRMED 상태이고 재고가 충분한 주문
- **When** 해당 주문에 대해 출고를 실행하면
- **Then** 주문 상태가 RELEASE로 전환된다.

### FR-21: 모니터링 — 주문량확인

시스템은 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 주문 목록을 확인할 수 있게 하며, REJECTED는 제외해야 한다. [SPEC §5-4]

- **Given** 각 상태의 주문이 존재할 때
- **When** 주문량확인을 실행하면
- **Then** RESERVED/CONFIRMED/PRODUCING/RELEASE 주문만 상태별로 표시되고 REJECTED 주문은 표시되지 않는다.

### FR-22: 모니터링 — 재고량확인

시스템은 각 시료별 현재 재고수량과, 해당 시료에 걸린 판단 대상 주문의 주문수 대비 여유/부족/고갈 상태를 표시해야 한다. [SPEC §5-4], [ADR-Q5]

- **Given** 특정 시료의 재고수량과 해당 시료에 대한 판단 대상 주문의 주문수가 주어졌을 때
- **When** 재고량확인을 실행하면
- **Then** 재고수량=0이면 "고갈", 0 < 재고수량 < 주문수이면 "부족", 재고수량 ≥ 주문수이면 "여유"로 표기된다.

### FR-23: 데이터 영속성

시스템은 애플리케이션을 재실행해도 등록된 시료와 주문 데이터를 유지해야 한다. [SPEC §7-영속성]

- **Given** 시료 및 주문 데이터가 저장된 상태에서 애플리케이션을 종료했을 때
- **When** 애플리케이션을 다시 실행하면
- **Then** 종료 전의 시료 및 주문 데이터가 그대로 유지되어 조회 가능하다.

## 5. 계산 규칙

### 5.1 수율

```
수율 = 정상적인 시료 / 총 생산시료
예: 100개 생산 중 정상품 90개 → 수율 = 0.9
```
[SPEC §4]

**허용 범위**: (0, 1] — 0 이하 또는 1 초과 값은 등록 시점에 거부된다. [ADR-Q4]

### 5.2 부족분

```
부족분 = max(0, 주문수 − 재고수량)
```
[ADR-Q1]

**경계 케이스**: 재고수량 ≥ 주문수이면 부족분 = 0.

### 5.3 실생산량

```
실생산량 = ceil(부족분 / 수율)
```
[SPEC §4], [ADR-Q2]

**경계 케이스**:
- 부족분 = 0이면 실생산량 = ceil(0 / 수율) = 0 — 생산 불필요.
- 수율 = 0인 경우는 등록 단계에서 시료 자체가 거부되므로(§5.1) 실생산량 계산에서 0으로 나누기 상황은 발생하지 않는다. [ADR-Q3]

### 5.4 총생산시간

```
총생산시간 = 평균생산시간 × 실생산량
```
[SPEC §4], [ADR-Q2]

화면 표기 시 곱셈 기호는 "×"를 사용한다. [ADR-Q16]

### 5.5 생산완료 시 재고 반영량

```
재고 반영량 = floor(실생산량 × 수율)
```
[ADR-Q13]

**경계 케이스**: ceil 연산의 특성상 실생산량 × 수율은 부족분과 같거나 그보다 약간 큰 값이 된다(예: 부족분 100, 수율 0.9 → 실생산량 = ceil(100/0.9) = 112 → 재고 반영량 = floor(112 × 0.9) = floor(100.8) = 100).

### 5.6 재고 상태 판정(여유/부족/고갈)

```
고갈: 재고수량 = 0
부족: 0 < 재고수량 < 주문수
여유: 재고수량 ≥ 주문수
```
[SPEC §4, §5-4], [ADR-Q5]

**경계 케이스**: 재고수량 = 주문수인 경우는 "여유"로 판정한다(재고수량 ≥ 주문수 조건 포함).

## 6. 비범위 (Non-goals)

- **PRODUCING 상태 주문의 취소/거절**: 지원하지 않는다. [ADR-Q6]
- **RELEASE 상태 이후의 반품/재입고 처리**: 지원하지 않는다. RELEASE는 종단 상태이다. [ADR-Q8]
- **REJECTED 상태로부터의 재전이(복구 등)**: 지원하지 않는다. REJECTED는 종단 상태이다. [ADR-Q8]
- **멀티 프로세스/멀티유저 동시 접근에 대한 방어(분산 락 등)**: 지원 범위 밖. 단일 프로세스 순차 처리만 전제한다. [ADR-Q10]
- **다중 생산라인/병렬 생산**: 지원하지 않는다. 생산큐는 단일 라인 순차 처리로만 동작한다. [ADR-Q11]
- **주문ID의 전역 고유성 보장(UUID 등)**: 자동증가 정수 채번으로 충분하며 별도 전역 고유 식별 체계는 도입하지 않는다. [ADR-Q15]
- **Order(주문)의 하드 삭제**: 지원하지 않는다. 어떤 상태의 Order도 물리적으로 제거할 수 없다 — REJECTED/RELEASE 전이가 논리적 삭제 역할을 하며, FR-32의 감사 추적 목적과 상충한다. [ADR-C3]

## 7. 추가 보강 개발 항목 (RISKS.md 기반)

> 본 절은 SPEC/DECISIONS에는 근거가 없으나 `docs/RISKS.md`의 리스크 분석(생산관리 시스템 실무 관점)에서 도출된 방어적 구현 항목이다.
> SPEC 근거가 없으므로 각 항목은 `[RISK §n]`으로 출처를 표기하며, 정식 기능요구가 아닌 "구현 시 반영 권장" 항목으로 구분한다.

### FR-24: 단일 인스턴스 가드

시스템은 이미 실행 중인 인스턴스가 있을 경우 두 번째 실행을 감지하고 경고 후 종료해야 한다. [RISK §1]

- **Given** 프로그램이 이미 하나 실행 중인 상태에서
- **When** 동일 데이터 저장소를 대상으로 프로그램을 다시 실행하면
- **Then** 두 번째 실행은 경고 메시지를 표시하고 즉시 종료한다(Named Mutex 또는 파일 배타 잠금 방식 권장).

### FR-25: 상태전이·재고증감의 원자적 처리

시스템은 주문 상태 전이와 재고 증감을 하나의 논리적 단위로 처리하여, 처리 도중 예외가 발생해도 재고수량과 주문 상태가 불일치한 채로 저장되지 않도록 해야 한다. [RISK §2]

- **Given** 재고 차감/증가와 주문 상태 전이가 함께 일어나는 처리(T1~T4, T6) 도중
- **When** 처리 중간에 예외가 발생하면
- **Then** 재고수량과 주문 상태 모두 처리 이전 값으로 유지되고(부분 반영 없음), 변경사항은 저장되지 않는다.

### FR-26: 수율 계산의 정밀도 보장

시스템은 수율 및 이를 사용한 실생산량/재고 반영량 계산에 부동소수점(`double`) 대신 고정소수점(`decimal`) 연산을 사용해야 한다. [RISK §4]

- **Given** 부족분과 수율이 정확히 나누어떨어지는 경계값(예: 부족분 90, 수율 0.9)일 때
- **When** 실생산량(ceil(부족분/수율))을 계산하면
- **Then** 부동소수점 오차 없이 정확한 정수 결과(예: 100)가 산출된다.

### FR-27: 지원하지 않는 전이 시도 시 명시적 거부 메시지

시스템은 PRODUCING 상태 주문에 대한 거절 시도, REJECTED/RELEASE 상태 주문에 대한 모든 전이 시도에 대해 조용히 무시하지 않고 명시적인 거부 메시지를 표시해야 한다. [RISK §5], (FR-13 보강)

- **Given** PRODUCING 상태의 주문에 거절을 시도하거나, REJECTED/RELEASE 상태의 주문에 임의의 전이를 시도했을 때
- **When** 해당 명령을 실행하면
- **Then** "지원하지 않는 처리입니다" 등 명확한 사유를 포함한 에러 메시지가 표시되고 상태는 변경되지 않는다.

### FR-28: 생산완료 처리 트리거 확정 (Q17)

시스템은 PRODUCING → CONFIRMED 전이(생산완료)를 발생시키는 트리거를 명시적으로 정의해야 한다. 콘솔 앱의 블로킹 입력 구조상 백그라운드 타이머 대신 담당자의 수동 명령(또는 틱 진행 명령)으로 전이를 실행한다. [RISK §6]

- **Given** 생산큐 선두에 PRODUCING 상태 항목이 있을 때
- **When** 생산담당자가 "생산완료 처리"(또는 "생산진행") 명령을 실행하면
- **Then** 해당 항목이 CONFIRMED로 전이되고 재고수량에 floor(실생산량 × 수율)만큼 반영된다.

### FR-29: 시료ID 참조 무결성 검증

시스템은 시료예약 시 입력된 시료ID가 실제 등록된 시료인지 검증하고, 시료등록 시 시료ID 중복 여부를 검증해야 한다. [RISK §7]

- **Given** 등록되지 않은 시료ID로 시료예약을 시도하거나, 이미 등록된 시료ID로 시료등록을 시도했을 때
- **When** 해당 명령을 실행하면
- **Then** 처리가 거부되고 명확한 에러 메시지("시료ID를 찾을 수 없습니다" / "이미 등록된 시료ID입니다")가 표시된다.

### FR-30: 영속성 저장 실패 복구 전략

시스템은 저장 시 임시 파일 작성 후 원자적 교체(atomic rename/`File.Replace`) 방식을 사용하고, 로드 시 파일 손상이 감지되면 백업본으로 폴백하거나 명확한 경고와 함께 새로 시작해야 한다. [RISK §8]

- **Given** 저장 도중 프로세스가 비정상 종료되었거나 저장 파일이 손상된 상태에서
- **When** 애플리케이션을 재실행하면
- **Then** 기존 정상 데이터는 보존되며(임시파일 교체 방식), 손상이 감지된 경우 백업본 로드 또는 "저장 파일이 손상되어 새로 시작합니다" 경고 후 새 상태로 시작한다.

### FR-31: 시료검색의 숫자 필드 일치 정책 확정

시스템은 시료검색(FR-05) 시 수율·평균생산시간과 같은 숫자 속성에 대해 정확 일치를 기본 정책으로 채택하고 이를 문서화해야 한다. [RISK §9]

- **Given** 수율 또는 평균생산시간 값을 검색어로 입력했을 때
- **When** 시료검색을 실행하면
- **Then** 해당 숫자 값과 정확히 일치하는 시료만 결과로 표시된다(범위 검색은 본 버전의 범위에서 제외).

### FR-32: 감사 추적을 위한 타임스탬프 기록

시스템은 각 주문의 생성 시각과 상태 전이 시각을 내부적으로 기록해야 한다(화면 표시 의무는 없음). [RISK §10]

- **Given** 주문이 생성되거나 상태가 전이될 때
- **When** 해당 이벤트가 발생하면
- **Then** 생성 시각(CreatedAt) 및 전이 시각이 함께 기록되어 추후 조회·디버깅에 활용 가능하다.

### FR-33: 시료 삭제 (CONTRACT 감사 기반)

시스템은 등록된 시료를 삭제할 수 있어야 하며, 해당 시료ID를 참조하는 Order가 하나라도(상태 무관, REJECTED/RELEASE 이력 포함) 존재하면 삭제를 거부해야 한다. [ADR-C1][ADR-C2]

- **Given** 어떤 Order도 참조하지 않는 등록된 시료ID
- **When** 해당 시료 삭제를 실행하면
- **Then** 시료가 저장소에서 제거되고 이후 조회에 나타나지 않는다.

- **Given** 하나 이상의 Order(상태 무관)가 참조하는 등록된 시료ID
- **When** 해당 시료 삭제를 시도하면
- **Then** 삭제가 거부되고 명확한 거부 메시지가 표시되며 시료는 그대로 유지된다.
