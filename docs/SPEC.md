# SPEC

## 1. 배경

가상의 반도체 회사 "S-Semi"가 있다.

이 회사는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품하고 있다.

시료는 주문이 들어오면 웨이퍼 공정설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

그런데 최근 들어 주문량이 급증하면서 문제가 생겼다.

- "어, 이 주문 처리됐나요?"
- "공정 예약을 했는데, 언제 완성되는지 모르겠어요."
- "이미 충분한 시료 재고가 있는데, 왜 추가 공정이 돌아가고 있나요?"

엑셀과 메모장으로 주문을 관리하다 보니, 실수가 잦고 재고와 공정현황을 한눈에 파악하기 어려웠다.

이러한 이유로 S-Semi에서는 더 체계적인 시료 관리를 위한 "반도체 시료 생산 주문관리 시스템"을 개발하기로 결정했다.

## 2. 역할 3종과 흐름

| 역할 | 내부/외부 | 담당 | 세부 내용 |
|---|---|---|---|
| 고객 | 외부 | 시료요청자 | 필요한 시료를 이메일로 요청 |
| 주문담당자 | 내부 | 주문서관리 | 요청에 맞게 주문서 작성, 생산담당자에게 주문서를 전달함 |
| 생산담당자 | 내부 | 시료생산, 승인 | 개발시료등록, 주문수신 후 승인 또는 거절 |

## 3. 주문 상태 흐름

모든 주문은 아래의 상태를 보유한다.

REJECTED는 거절된 주문으로 정상흐름 외의 상태이며 모니터링에서 제외된다.

| 상태 | 의미 |
|---|---|
| RESERVED | 주문접수 |
| REJECTED | 주문거절 |
| PRODUCING | 주문승인 완료 및 재고부족으로 생산중 |
| CONFIRMED | 주문승인 완료 및 출고 대기중 |
| RELEASE | 출고완료 |

### 접수된 주문 목록

- RESERVED 상태의 주문 목록 Display
- 특정 주문에 대하여 승인 혹은 거절

### 주문승인

- 접수된 특정 주문에 대해 승인
- 승인시 재고상황에 따라 2가지 방식으로 자동으로 처리
  - 재고가 충분한 경우 → 주문을 즉시 CONFIRMED 상태로 전환
  - 재고가 부족한 경우 → 생산라인에 자동으로 등록, 주문상태를 PRODUCING으로 전환

### 주문거절

- 접수된 특정 주문에 대해 거절
- 즉시 REJECTED 상태로 전환

### 출고처리

- 재고가 충분해진 CONFIRMED 주문에 대하여 출고를 처리
- 특정 주문에 대해 출고를 실행
- 주문상태가 RELEASE로 전환

### 생산처리

- 생산라인에 대한 정보를 Display
- 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산
- 실생산량 = ceil(부족분 / 수율)
- 총생산시간 = 평균생산시간 * 실생산량
- 생산완료시 주문상태 PRODUCING → CONFIRMED 변경

## 4. 계산수식

> 수식은 한 글자도 틀리면 안 됨. 수정 시 별도 검증 세션 진행 필요함.

```
수율 : (정상적인 시료/ 총 생산시료)
ex) 100개 생산 중 정상적인 물품90개 = 0.9

실생산량: ceil(부족분 / 수율)

총생산시간: 평균생산시간* 실생산량

주문대비 재고수량에 따라 상태도 표기
- 여유: 주문대비 재고충분상태
- 부족: 주문대비 재고수량부족상태
- 고갈: 수량이 0인 상황
```

> [미정 - UI 섹션 반영 예정] 위 수식 중 "*"(곱셈)는 문서상 표기이며, UI에 표시할 때는 "×"로 표기해야 함. (사용자 지시: 나중에 UI 섹션에 명세)

### 검증: 원본 예시와 수식의 불일치

입력값: 주문량 300, 재고 30, 부족분 170, 수율 0.92, 평균생산시간 0.8
화면표기값(사용자 제시): 실생산량 206ea, 총생산시간 165min

| 항목 | 사용한 입력값 | 적용 수식 (SPEC.md §4) | 계산값 | 화면표기값 | 일치 여부 |
|---|---|---|---|---|---|
| 실생산량 | 부족분=170, 수율=0.92 | ceil(부족분 / 수율) | ceil(170 / 0.92) = ceil(184.7826...) = 185 | 206 | 불일치 |
| 총생산시간 | 평균생산시간=0.8, 실생산량(계산값)=185 | 평균생산시간 * 실생산량 | 0.8 * 185 = 148 | 165 | 불일치 |

[미정] 어느 쪽(수식 vs 화면표기값)이 정답인지는 여기서 판단하지 않음 — 사용자가 DECISIONS에서 결정.

### 1. 시료 관리

시료(Sample)는 이 시스템의 가장 기본이 되는 단위.
각 시료는 고유한 이름과 속성을 가지며, 시스템에 등록된 시료만 주문 가능.

| 메뉴 | 의미 |
|---|---|
| 시료등록 | 새로운 시료를 시스템에 추가, 속성값: 시료ID, 이름, 평균 생산시간, 수율 |
| 시료조회 | 등록된 모든 시료 목록을 확인, 현재 재고수량도 함께 표시 |
| 시료검색 | 등록된 모든 시료 목록을 확인, 현재 재고수량도 함께 표시, 이름 등 속성으로 특정 시료를 검색 |

```
* 수율 : (정상적인 시료/ 총 생산시료)
ex) 100개 생산 중 정상적인 물품90개 = 0.9
```

### 2. 시료 주문

고객이 시료를 요청하면 주문담당자가 주문을 생성.

#### 시료예약
- 고객이 원하는 시료와 수량을 주문
- 이 시점에서 주문상태는 RESERVED
- 예약시 입력값: 시료ID, 고객명, 주문수

### 3. 주문 승인/거절

접수된 주문(RESERVED) 목록을 확인.
특정 주문에 대하여 승인 혹은 거절.

#### 접수된 주문목록
- RESERVED 상태의 주문 목록 Display

#### 주문승인
- 접수된 특정 주문에 대해 승인
- 승인시 재고상황에 따라 2가지 방식으로 자동으로 처리
  - 재고가 충분한 경우 → 주문을 즉시 CONFIRMED 상태로 전환
  - 재고가 부족한 경우 → 생산라인에 자동으로 등록, 주문상태를 PRODUCING으로 전환

#### 주문거절
- 접수된 특정 주문에 대해 거절
- 즉시 REJECTED 상태로 전환

### 4. 모니터링

주문에 따른 시료 재고에 대한 정보 모니터링 기능.
담당자가 현재 시스템의 상태를 한눈에 파악할 수 있도록 구성.

#### 주문량확인
- 현재 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 목록을 확인
- REJECTED는 유효한 주문이 아니므로 무시

#### 재고량확인
- 각 시료별 현재 재고수량을 확인
- 주문대비 재고수량에 따라 상태도 표기
  - 여유: 주문대비 재고충분상태
  - 부족: 주문대비 재고수량부족상태
  - 고갈: 수량이 0인 상태

### 5. 생산 라인

- 생산라인에 대한 정보를 Display
- 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산
- 실생산량: ceil(부족분 / 수율)
- 총생산시간: 평균생산시간* 실생산량 [해석: UI 표기 시 §4 하단 메모대로 "×"로 표시]
- 생산완료시 주문상태 PRODUCING -> CONFIRMED 변경

#### 생산현황표기
- 현재 생산중인 시료에 대한 정보 표기
- 표기할 정보 수준은 [미정 - 자율적으로 결정하도록 명시됨] (ex) 주문 정보, 현재까지의 생산량 등

#### 대기주문확인
- 생산라인의 대기열인 생산큐를 이용
- 생산작업을 대기하고 있는 목록을 출력
- 스케줄링 전략: FIFO

### 6. 출고 처리

- 재고가 충분해진 CONFIRMED 주문에 대하여 출고를 처리
- 특정 주문에 대해 출고를 실행
- 주문상태가 RELEASE로 전환

## 7. 제약

콘솔, 영속성, UI 자유

### 콘솔
- 시스템은 콘솔기반으로 동작
- 담당자가 직접 명령을 입력
- 시료를 등록하고 주문을 처리

### 영속성
- 데이터 영속성: application을 다시 실행해도 데이터를 유지할 수 있는 성질

### UI
- UI는 자유롭게 구성해도 됨
- 단, 콘솔에서 한눈에 들어오는 것을 고려할 것
- 최대한 생산관리프로그램답게 구성할 것
